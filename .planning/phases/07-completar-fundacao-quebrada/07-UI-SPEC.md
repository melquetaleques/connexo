## Component Interaction Contracts

This section is Phase 7-specific. It defines the exact states for each wired interaction.

### 1. ServicesPage — "Cadastrar Serviço" Form

**Trigger:** GoldButton "Cadastrar serviço" (icon: `add`) opens inline card panel (already exists as `editing` state).

**Form fields (all required):**
- Título do serviço — `Field`, placeholder "Ex: Perícia Contábil Judicial"
- Descrição — `<textarea>` (rows=3), placeholder "Descreva o serviço..."
- Valor a partir de (R$) — `Field`, placeholder "1.800,00", type text (formatted input)
- Prazo médio (dias) — `Field`, placeholder "30", type number

**Submit button:** GoldButton (type="submit") label "Salvar serviço" with icon `check`.

**States:**

| State         | Visual Contract                                                                 |
|---------------|---------------------------------------------------------------------------------|
| Idle          | GoldButton enabled, no feedback                                                 |
| Loading       | GoldButton `disabled`, icon replaced by `autorenew` spinning (text: "Salvando...")|
| Success       | Panel closes. New service card appears in the grid with a brief gold ring flash (ring-2 ring-secondary/40 animate once). No toast — the card itself is confirmation. |
| Error (4xx)   | Panel stays open. Below form footer, inline error text: `text-rose-600 text-xs font-bold` — "Erro ao salvar serviço. Verifique os campos e tente novamente." |
| Error (network)| Same visual as 4xx. Text: "Sem conexão com o servidor. Tente novamente." |
| Duplicate name | 400 from API → same inline error bar. Text: "Já existe um serviço com este título." |

**Empty state (no services yet):**
- Container: `rounded-[32px] border-2 border-dashed border-outline/30 bg-surface py-20 text-center`
- Icon: `business_center` at `text-4xl text-primary/10 mb-4`
- Heading: `"Nenhum serviço cadastrado"` — `text-sm font-bold text-primary/30 uppercase tracking-widest`
- Body: `"Clique em 'Cadastrar serviço' para adicionar sua primeira oferta."` — `text-xs text-primary/20 mt-2`

Source: Empty state container pattern already exists in `ServicesPage.tsx` — add body line only.

### 2. PostsPage — "Publicar" Button

**Trigger:** GoldButton "Nova postagem" (icon: `add`) opens inline compose card (already exists as `composing` state).

**Form fields (all required per D-10):**
- Cover image — `StripePlaceholder` with click-to-upload behavior (Phase 7 uploads go to MinIO via `/api/acc/postagens` which accepts `cover_url`; for MVP, accept a URL text input if file upload is deferred — see note below)
- Título — `Field`, placeholder "Sobre o que você quer escrever?"
- Resumo — `<textarea>` (rows=3), placeholder "Resumo curto que aparece no card..."
- Categoria — tag chip selector (existing TAGS array): `["Tributário", "Empresarial", "Trabalhista", "Carreira", "Sucessões", "Cível"]`
- Conteúdo — `<textarea>` (rows=8), label "Conteúdo", placeholder "Escreva o conteúdo completo do artigo..."

> **Cover URL note (MVP):** MinIO media upload is Phase 9 (PROF-01). For Phase 7, replace `StripePlaceholder` with a `Field` input for "URL da capa" (type="url", placeholder "https://..."). The `StripePlaceholder` stays as a visual preview when a valid URL is entered.

**Selected tag state:** Active tag chip uses `bg-secondary/10 border-secondary/40 text-secondary` (gold tint) instead of default `border-outline text-on-surface-variant`.

**Submit button:** GoldButton (type="submit") label "Publicar" with icon `publish`. The existing "Agendar" GhostButton is removed — per D-09, no draft/schedule workflow in MVP.

**States:**

| State          | Visual Contract                                                                  |
|----------------|----------------------------------------------------------------------------------|
| Idle           | GoldButton enabled                                                               |
| No tag selected| GoldButton disabled (`disabled` prop) — "Publicar" grayed — until at least one tag is selected |
| Loading        | GoldButton `disabled`, icon `autorenew` spinning, text "Publicando..."           |
| Success        | Compose panel closes. New post appears at position 0 in the grid/feed. No toast. |
| Error (4xx)    | Panel stays open. Inline error below footer: `text-rose-600 text-xs font-bold` "Erro ao publicar. Verifique os campos e tente novamente." |
| Error (network)| Same container: "Sem conexão. Tente novamente."                                  |

**Empty state (no posts yet):**
- Container: `rounded-[32px] border-2 border-dashed border-outline/30 bg-surface py-20 text-center`
- Icon: `edit_note` at `text-4xl text-primary/10 mb-4`
- Heading: `"Nenhuma postagem ainda"` — `text-sm font-bold text-primary/30 uppercase tracking-widest`
- Body: `"Compartilhe seu conhecimento. Clique em 'Nova postagem' para começar."` — `text-xs text-primary/20 mt-2`

### 3. UsersPage — "Convidar Usuário" Flow

**Trigger:** GoldButton "Convidar usuário" (icon: `person_add`) opens an inline modal-style Card panel below the page header. This is NOT a full modal overlay — it follows the same pattern as ServicesPage and PostsPage (inline expanding card).

**Panel placement:** Inserted between the page header and the users table Card, same as `editing` card in ServicesPage.

**Form fields (all required):**
- E-mail do convidado — `Field`, type="email", placeholder "nome@escritorio.com.br", label "E-mail"
- Nome — `Field`, type="text", placeholder "Nome completo do convidado", label "Nome"
- Permissão — `SelectField`, label "Nível de acesso", options:
  - `{ label: "Administrador", value: "admin" }`
  - `{ label: "Editor", value: "editor" }`
  - `{ label: "Visualizador", value: "viewer" }`

**Submit button:** GoldButton (type="submit") label "Enviar convite" with icon `send`. Next to it: GhostButton "Cancelar" closes the panel.

**States:**

| State         | Visual Contract                                                                  |
|---------------|----------------------------------------------------------------------------------|
| Idle          | GoldButton enabled                                                               |
| Loading       | GoldButton `disabled`, icon `autorenew` spinning, text "Enviando..."             |
| Success       | Panel closes. Inline success notice appears at the top of the users table: `rounded-lg border border-emerald-200/60 bg-emerald-50 px-4 py-3 text-emerald-700 text-sm font-bold` — "Convite enviado para {email}. O link expira em 72 horas." Dismiss with `Icon name="close" aria-label="Fechar"` button after 6 seconds or on click. |
| Error (409 — already member) | Panel stays open. Inline error: "Este e-mail já é membro do escritório." |
| Error (429 — rate limit)     | Panel stays open. Inline error: "Muitos convites enviados. Aguarde antes de tentar novamente." |
| Error (network)| Panel stays open. Inline error: "Sem conexão com o servidor. Tente novamente." |

**Empty state (no team members — unlikely but possible):**
- Container: same dashed border pattern as ServicesPage empty state
- Icon: `group` at `text-4xl text-primary/10 mb-4`
- Heading: `"Equipe vazia"` — `text-sm font-bold text-primary/30 uppercase tracking-widest`
- Body: `"Convide membros do escritório para acessar o portal."` — `text-xs text-primary/20 mt-2`

> Note: The existing `UsersPage.tsx` renders the table unconditionally. Add a conditional check: `users.length === 0` renders empty state; otherwise renders the table Card.

### 4. AccountantCatalogPage — Filter UX

**Current state:** Filter bar exists with 4 fields (q, city, specialty, state). Filters debounce 400ms and call `listAccountants(filters)`. The API currently ignores filter params (returns all). Phase 7 wires the backend SQL.

**No structural changes to filter bar layout.** Only the active-filter visual feedback is new.

**Active filter chip indicator:**

When any filter field is non-empty (value !== ""), add a gold dot indicator to the "Filtros Avançados" GhostButton label:

```
Filtros Avançados  [●] 2
```

- Replace the `filter_list` icon with `filter_list_off` when filters are active
- Append a count badge: `<Pill tone="gold">{activeCount}</Pill>` inline after the button text
- This is a read-only count — the button does not open a panel in Phase 7

**"Limpar filtros" link:** When `activeCount > 0`, show a `text-xs font-bold text-secondary underline cursor-pointer` link "Limpar filtros" immediately to the right of the filter bar row. Clicking it resets all filters to `{ q: "", specialty: "", city: "", state: "" }`.

**Loading state during filter change:**
- Grid area replaces with the existing loading spinner (icon `autorenew` + text "Buscando especialistas...")
- Filter fields remain interactive during loading (do not disable them)

**Empty state (no results for active filter):**
- Existing empty state is preserved: `search_off` icon + "Nenhum perito encontrado" heading
- Add a body line: `"Tente remover alguns filtros para ampliar a busca."` — `text-primary/50 font-medium`
- Add a GhostButton "Limpar filtros" centered below the body copy, which resets all filters

**Empty state (no filters, no accountants — zero data):**
- Icon: `group_off` at `text-3xl text-primary/20`
- Heading: `"Nenhum perito cadastrado ainda"` — `text-xl font-black text-primary`
- Body: `"O catálogo está sendo populado. Volte em breve."` — `text-primary/50 font-medium`

## Interaction Patterns — Phase 7 Standard

These patterns apply uniformly to all three wired forms (ServicesPage, PostsPage, UsersPage):

### Loading Pattern
GoldButton `disabled` prop set to `true`. Children rendered as:
```
<Icon name="autorenew" className="text-base animate-spin" /> {loadingLabel}
```
The `disabled` state already handles `opacity-50 cursor-not-allowed scale-100` via connexo-primitives `GoldButton`.

### Inline Error Pattern
A single `<div>` rendered below the form footer border-top, visible only when `error !== null`:
```
rounded-lg border border-rose-200/60 bg-rose-50 px-4 py-3 text-rose-700 text-xs font-bold
```
Error clears on next submit attempt or on input change (form `onChange` handler sets `setError(null)`).

### Inline Success Pattern (UsersPage only — invite)
A `<div>` rendered above the users table, visible only when `successMsg !== null`, auto-dismissed after 6s:
```
rounded-lg border border-emerald-200/60 bg-emerald-50 flex items-center justify-between px-4 py-3 text-emerald-700 text-sm font-bold
```
Dismiss button: `Icon name="close"` at right edge, `onClick={() => setSuccessMsg(null)}`.

### Form Reset on Success
All wired forms: on successful API response, reset form state (`setFormData(initial)`) and close panel (`setEditing(false)` / `setComposing(false)` / `setInviting(false)`). Refresh the list by calling the load function again.
